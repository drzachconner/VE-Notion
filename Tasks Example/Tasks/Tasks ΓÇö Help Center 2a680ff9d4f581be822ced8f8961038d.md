### Quick start

1. Add your first tasks in [Inbox](Inbox%202a680ff9d4f581d1b8d6ede1d4a96eaf.md)
2. Set Status and Due Date
3. Check [Today](Today%202a680ff9d4f5810e9640dcfcd9daa9ac.md) and [Week](Week%202a680ff9d4f5816e9cc9ead984d36616.md) to plan
4. Use Tag and Priority for context and importance
5. Mark tasks Done to archive them in Logbook

### Properties explained

- Name: Task title
- Status: Inbox, To-Do, Doing, Done
- Due Date: Date used by Today, Week, Month views
- Tag: Context such as Home, Work, Personal, Business, Project, Events, Research, Study, Health, Finance, Fitness, Shopping, Travel, Family
- Priority: High, Medium, Low
- Notes: Rich details for the task
- Recurring controls:
    - Recurring Interval: Number of days for spacing
    - Recurring Unit: Day(s), Week(s), Month(s), special monthly rules, Year(s)
    - Recurring Days: If unit is Week(s), pick weekdays
    - Next Due Date, Past Due Date, Recurring: computed helpers
- Parent Task, Task History: Relations for nesting and audit trail

### Views overview

- Tables
    - All: Everything due today or later, plus no date, excluding Inbox
    - Today: Due Date is today
    - Week, Month: Upcoming schedule, sorted by Due Date
    - Anytime: No Due Date and not Inbox or Done
    - Recurring: Future dated and has a recurring interval
    - Past Due: Overdue items
    - Inbox: Intake view
- Boards
    - Tag board: Grouped by Tag for context switching
    - Board - Status and Board - Priority: Kanban by workflow or importance
- Calendars
    - Week and Month calendars on Due Date
- Chart
    - Status - Visual: Count by Status

### Common workflows

- Capture
    - Add new items in Inbox with minimal fields
- Plan today
    - Set Due Date to today or drag into Today
    - Move status to To-Do or Doing
- Schedule week
    - Assign Due Dates across the week and set Priority
- Recurring tasks
    - Set Recurring Unit and Interval
    - For weekly patterns, set Recurring Days
    - The system computes Next Due Date automatically
- Review and close
    - Mark Done to move to Logbook

### Tips

- Keep Inbox for untriaged items only
- Use Priority for sequencing, Tag for context
- Filter by Tag when working in a specific area
- Keep Due Date realistic to reduce overdue noise

### Troubleshooting

- A recurring task didn’t advance
    - Ensure Recurring Unit and Interval are set
    - For weekly cadence, set Recurring Days
    - Check that Stop Recurring isn’t pressed
- Items not showing in Today
    - Confirm Due Date is today and Status is not Inbox
- Overdue tasks not visible in Past Due
    - Verify Due Date is in the past and Status is not Done
